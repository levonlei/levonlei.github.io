golang中下划线_的用法

#golang中下划线_的用法

目录

  • 用法一：类型断言
  • 用法二：忽略返回值
  • 用法三：引入包

背景

在看很多源码的时候会出现下划线的用法，尤其是用在接口与结构体之间进行断言，kubernets，kuberbuilder之中都存在。自己总结了一下，下划线的用法，记录下来。

用法一：类型断言

假设我们定义了interface 
type Foo interface {
     Say()
}
定义struct 
type Dog struct {
}
后我们希望在代码中判断Dog这个struct是否实现了Foo这个interface，就可以如下这么写。 
var _ Foo = Dog{} {
}
假设没有实现的话，ide会波浪线标识以及告诉你哪个没实现，再者编译的时候会直接报错。可以很好的帮助我们编程时候发现错误

用法二：忽略返回值

假设一个函数定义了多个变量，我们只需要其中一个，其他的不要用到，就可以这么使用。这是下划线在go中最常见的用法了。 
v1, _ := func(...) {
}

用法三：引入包

假设我们在代码的import中这样引入package： 
import _ "test/foo" {
}
这表示呢在执行本段代码之前会先调用test/foo中的初始化函数(init)，这种使用方式仅让导入的包做初始化，而不使用包中其他功能。适用于本包文件的运行，必须执行某个包的init函数，避免panic问题
